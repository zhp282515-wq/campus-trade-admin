# 项目名称

校园二手交易后台管理系统 / Campus Trade Admin

# 开发策略

本项目分两个版本：

1. 版本 1：先做前端 + Mock 数据，不接后端，不接 MySQL。
2. 版本 2：前端完成后，再接 Express + MySQL，把 Mock 数据替换成真实接口。

# 技术栈

前端：

Vue3 + Vite + JavaScript + Vue Router + Pinia + Element Plus + ECharts + Axios

后端：

版本 1 暂不实现后端。

版本 2 再使用 Node.js + Express + MySQL。

# 开发规则

1. 使用 JavaScript，不使用 TypeScript。
2. 代码要简单清晰，适合初学者理解。
3. 不要一次性生成整个项目，按模块逐步实现。
4. 不要随意删除已有文件。
5. 不要执行危险命令。
6. 版本 1 所有数据都放在 `src/mock` 目录。
7. 版本 1 页面只调用 `src/api` 中封装的方法，不要在页面里直接写死数据。
8. `src/api` 第一版可以从 mock 文件读取数据，第二版再改成 axios 请求后端接口。
9. 页面要有 loading、空数据状态、操作成功/失败提示。
10. 每完成一个模块，都要给出测试步骤。
11. 所有时间字段统一精确到秒，格式使用 `YYYY-MM-DD HH:mm:ss`。

# 运行命令

前端启动：

```bash
npm run dev
```

# UI 设计规范

本项目的 UI 风格参考“校园二手交易后台管理系统 / 数据看板”设计图。后续实现时应尽量贴近参考图的整体气质，但不要求像素级完全一致。

### 1. 整体风格

- 采用现代、简洁、清晰的后台管理系统风格。
- 页面应突出管理效率，避免复杂装饰和营销式布局。
- 信息层级要明确，常用功能要容易找到。
- 整体视觉应偏清爽、规整，适合校园二手交易管理场景。

### 2. 主色调

- 左侧侧边栏使用深蓝色或深蓝黑色。
- 主要按钮、菜单高亮、链接操作使用蓝色。
- 内容区域以白色卡片为主。
- 页面背景使用浅灰色或浅灰蓝色。
- 状态颜色保持统一：
  - 正常、启用、成功：绿色
  - 待处理、警告：橙色
  - 禁用、失败、驳回、危险操作：红色
  - 辅助信息：灰色

### 3. 布局规范

- 后台页面采用左侧固定侧边栏、顶部导航栏、右侧内容区的结构。
- 左侧侧边栏用于展示系统名称和功能菜单。
- 顶部导航栏用于展示折叠菜单按钮、面包屑、通知图标、用户信息等。
- 内容区使用卡片式布局承载统计数据、图表、搜索区域和表格。
- 页面模块之间保持统一间距，避免内容过于拥挤。
- 内容区背景使用浅灰色，卡片背景使用白色。

### 4. 登录页规范

- 登录页不显示后台侧边栏和顶部导航栏。
- 登录卡片居中展示。
- 背景使用校园插画风格或浅蓝色校园场景风格。
- 登录卡片包含系统图标、系统名称、副标题、用户名输入框、密码输入框和登录按钮。
- 输入框使用 Element Plus 表单组件风格。
- 登录按钮使用蓝色主按钮，宽度与输入框保持一致。

### 5. 数据看板规范

- 数据看板顶部展示统计卡片。
- 统计卡片可包含图标、指标名称、指标数值和变化趋势。
- 图表区域包含折线图、饼图或环形图、柱状图。
- 折线图用于展示订单趋势或成交趋势。
- 饼图或环形图用于展示分类占比、商品状态等比例数据。
- 柱状图用于展示学院发布商品数量等对比数据。
- 卡片保持适度圆角和轻微阴影，圆角不宜过大。
- 图表配色应包含蓝色、绿色、橙色、紫色等辅助色，避免整页只有单一蓝色。

### 6. 表格页规范

- 表格页结构保持统一：顶部搜索筛选区域，中间数据表格，底部分页。
- 搜索筛选区域使用输入框、下拉框、查询按钮、重置按钮等 Element Plus 组件。
- 数据表格使用 Element Plus 表格组件。
- 表头使用浅灰背景，表格内容保持清晰紧凑。
- 底部分页使用 Element Plus 分页组件。
- 操作按钮使用文字按钮或小按钮。
- 普通操作使用蓝色，危险操作使用红色。
- 状态字段优先使用标签组件展示，并保持颜色含义统一。

### 7. 异常页规范

- 403 和 404 页面居中展示。
- 页面包含大号错误码、简短提示文案和返回首页按钮。
- 可加入简单插画或浅色装饰，但不要影响主要信息阅读。
- 返回首页按钮使用蓝色主按钮。
- 403 用于无权限访问，404 用于页面不存在。

### 8. 组件封装建议

- `AppSidebar`：后台左侧菜单组件。
- `AppHeader`：后台顶部导航栏组件。
- `PageContainer`：页面统一内容容器。
- `StatCard`：数据看板统计卡片组件。
- `SearchForm`：表格页搜索筛选区域组件。

组件封装应保持简单，优先服务当前项目需求。版本 1 阶段不要过度抽象，等多个页面出现明显重复后再继续拆分。
